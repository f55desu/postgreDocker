Команды, которые необходимо использовать для развертывания базы данных PostgreSQL с помощью Docker:

Dockerfile:
FROM postgres:latest                                - Указывает базовый образ, на основе которого будет создан контейнер.
ENV POSTGRES_PASSWORD=0000                          - Устанавливает переменную окружения POSTGRES_PASSWORD с значением "0000"
ENV POSTGRES_USER=f55                               - Устанавливает переменную окружения POSTGRES_USER с значением "f55"
ENV POSTGRES_DB=db_f55                              - Устанавливает переменную окружения POSTGRES_DB с значением "db_f55"
COPY init.sql docker-entrypoint-initdb.d/init.sql   - Копирует локальный файл init.sql внутрь контейнера и размещает его в директории docker-entrypoint-initdb.d
VOLUME /d_data:/var/lib/postgresql/data             - Создает том на хосте и монтирует его внутри контейнера

Команды терминала для запуска контейнера:

"docker build -t d_img:latest ." - используется для создания Docker-образа из Dockerfile.
1. -t: Этот флаг указывает тэг образа, через двоеточие указана версия этого образа. "latest" ставится по-умолчанию.
2. .: Указывает текущую директорию для Dockerfile

"docker run -d -p 5432:5432 --name d_cont -v /d_data:/var/lib/postgresql/data d_img:latest" - используется для создания и запуска контейнера на основе указанного Docker-образа.
1. -d: Этот флаг указывает Docker на запуск контейнера в фоновом режиме. Контейнер будет работать в фоне, и управление им будет осуществляться через другие команды.
2. -p 5432:5432: Этот флаг определяет проброс портов. Он связывает порт 5432 на хостовой машине с портом 5432 внутри контейнера. Это позволяет приложениям на хостовой машине подключаться к службе, работающей внутри контейнера, используя порт 5432.
3. --name d_cont: Этот флаг устанавливает имя контейнера в "d_cont". Имя контейнера позволяет обращаться к нему и управлять им с помощью команд Docker.
4. -v /d_data:/var/lib/postgresql/data: Этот флаг выполняет монтирование тома "/d_data" на хосте внутрь контейнера "/var/lib/postgresql/data".
5. d_img:latest: Это имя Docker-образа и его тег. "d_img" - это имя образа, а "latest" - это тег, указывающий на последнюю версию образа, которая будет использоваться для создания контейнера.

"docker exec -it d_cont psql -d db_f55 -U f55" - используется для выполнения команд внутри контейнера. В данном случае она выполняет команду psql внутри контейнера с именем d_cont.
1. -it: Эти флаги обозначают интерактивный режим (-i) и присоединение к терминалу (-t), что означает, возможность взаимодействовать с командой внутри контейнера и видеть её вывод в терминале.
2. d_cont: Это имя контейнера, в котором будет выполняться команда.
3. psql: Это команда, которую вы хотите выполнить внутри контейнера. В данном случае, это команда psql, которая используется для подключения к базе данных PostgreSQL.
4. -d db_f55: Этот флаг указывает базу данных, к которой вы хотите подключиться. db_f55 - это имя базы данных.
5. -U f55: Этот флаг указывает имя пользователя, с которым вы хотите подключиться к базе данных. f55 - это имя пользователя.
